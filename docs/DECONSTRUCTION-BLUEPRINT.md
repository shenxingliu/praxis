# Deconstruction Evolution Blueprint — from concept cards to a self-breeding aesthetic vocabulary

Date: 2026-07-05 · Goal: deconstruct references → re-fuse elements across references → distill taste → learn from the brand soul to create **new references** → serve concrete purposes.

Visual / Feeling / Communication are three **lenses**. To go deeper, four things are missing: **depth (the abstraction ladder), grammar (relations), coordinates (a taste profile), and reproduction (the synthesis loop)**.

## 1 · The abstraction ladder — a concept isn't a card, it's an elevator

Today a concept card = lens + concept + translation. Upgrade it to a four-level ladder; the same observation rides up and down:

```
L4 Worldview   what it believes        "truth needs shadow"
L3 Concept     the transferable idea   "revelation through concealment"     ← today's concept
L2 Principle   the formal device       "a single light source builds visual hierarchy"
L1 Percept     the literal visual fact "low warm side light, long shadows"  ← today's manifestation
```

**The value is in choosing the transfer level**: transferring at L1 = imitation (copying the lighting); at L3/L4 = creation (realizing the same belief through entirely different means). Fusion can specify "take L4 from this image, realize it with that image's L2" — that is true deconstruction, and it maps cleanly onto the L1–L5 artifact levels of the original N/S/I framework.

Implementation: decompose outputs all four levels per concept; recombination injects the text of whichever level was chosen.

## 2 · Relational decomposition — the grammar between concepts

A strong reference often works not through single concepts but through the **relations** between them:

- **Tension pairs**: raw material × refined light (the collision reads as premium)
- **Ratios**: 90% calm + 10% surprise (too much surprise becomes noise)
- **Dependencies**: that stillness only holds under a single diagonal composition

Decomposition additionally outputs `relations: [{kind: tension|ratio|dependency, a, b, note}]`. When fusing elements from different references, relations become a **compatibility grammar** — instead of shaking concepts in a bag, check "whose dependency does this combination break, what new tension does it create". The critic reuses these relations directly during review.

## 3 · Taste profile — soul is the "ought", taste is the "is"; the delta is the gold

The soul declares what the brand should be; every choice you make (calibration games, adoptions, rejections, keep/discard in fusion experiments) reveals what actually attracts you. Accumulate and cluster:

- **Taste attractors**: "you were drawn to 'containment' 7 times across 4 references from different sources" — preference patterns you yourself may not have noticed
- **Soul–taste delta report**: where the taste profile conflicts with the soul, either the soul should be updated (your brand is evolving) or it's a drift warning (you're straying from your own red-lines). The system proposes; you rule.
- **Purpose conditioning**: taste isn't global — hero-image taste ≠ social-image taste. Attractors are bucketed by purpose; the concept agent weights its picks by the current purpose.

## 4 · Fusion Lab — synthesizing new references (the self-breeding loop)

**This is the core you asked for.** New page / new flow:

```
Pick 2–4 concept cards (across references, across lenses, abstraction level selectable)
  → a synthesis agent generates a brand-new, purely aesthetic reference (no product! aesthetics only)
  → you judge: keep → into the pool (source: 'synthesized', low initial weight) / discard
  → kept syntheses are decomposed in turn — new vocabulary enters the library
  → the next generation can fuse the products of the last
```

The evolutionary structure is complete: **crossover** (recombination across images) + **mutation** (wildcard collisions) + **selection** (your verdicts + adoption rate) = an aesthetic evolution system whose fitness function is your taste. Original references are generation 0; a few cycles in, the vocabulary contains **visual language no single source ever had** — that is "creating new references".

Anti-degeneration guardrails: synthesized images carry a generation number (gen 1, gen 2…); every generation must pass the soul red-line check before entering the pool; a synthesis only gains weight when genuinely adopted (a product image made with it gets saved/exported) — preventing aesthetic inbreeding.

## 5 · Anti-taste — decomposition must also output repulsion

When decomposing a reference, also extract "what this image does that this brand must **never** do" (anti-concepts). Repulsion is half of taste. Anti-concepts feed straight into the critic's rule set, so reviews grow ever sharper about your red lines.

## Implementation order (suggested)

| Phase | Scope | Cost | Why this order |
|---|---|---|---|
| R1 | **Fusion Lab**: pick cards → synthesize a new reference + pool entry + re-decompose + generation tags | 1 day | Hits the core ask; playable and verifiable immediately |
| R2 | **Abstraction ladder**: decompose outputs four levels + fusion picks the transfer level | 0.5 day | Turns R1's fusion from imitation into creation |
| R3 | **Taste profile**: attractor clustering + soul–taste delta + purpose buckets | 1 day | Better once R1/calibration have accumulated some data |
| R4 | Relational grammar + anti-taste | 1 day | Quality lift; do it after R1 exposes real problems |
| R5 | Automated evolution (scheduled self-breeding, budget-capped) | later | Automate only after adoption rates prove synthesis quality |

## Boundaries

1. Synthesizing references costs real generation money — Fusion Lab runs on the flash model ($0.04/image) so evolution experiments don't hurt.
2. A taste profile needs ~20 choice signals minimum to mean anything; showing "not enough data yet" early is more honest than forcing conclusions.
3. Out of scope: cross-brand taste transfer, and fully automated evolution with no human judging (your verdict IS the fitness function; remove it and the system drifts).
