# Deconstruction 进化蓝图 — 从概念卡到自我繁殖的美学词汇库

日期：2026-07-05 · 目的：deconstruct reference → 跨 reference 重新融合 element → 提炼 taste → 依据 brand soul 学习创造出**新的 reference** → 服务具体目的

Visual / Feeling / Communication 是三个**视角（lens）**。要往深处开发，还缺四样东西：**深度（抽象阶梯）、语法（关系）、坐标（品味画像）、繁殖（合成回路）**。

## 一、抽象阶梯 — 每个概念不是一张卡，是一部电梯

现在一张概念卡 = 视角 + 概念 + 转译。升级为四层抽象阶梯，同一观察沿梯子上下：

```
L4 Worldview   它相信什么      "真实需要阴影"
L3 Concept     可迁移的想法    "以遮蔽完成揭示"          ← 现在的 concept
L2 Principle   起作用的形式原理 "单一光源制造视觉等级"
L1 Percept     字面视觉事实    "低角度暖色侧光，长投影"   ← 现在的 manifestation
```

**关键价值在迁移层级的选择**：在 L1 迁移 = 模仿（抄光线）；在 L3/L4 迁移 = 创造（用完全不同的手段实现同一个信念）。融合时可以指定"从这张图取 L4、用那张图的 L2 去实现"——这是真正的 deconstruction，而且正好呼应你最初 N/S/I 框架里的 L1-L5 artifact 层级。

实现：decompose 输出每个概念的四层；重组注入时按所选层级取文本。

## 二、关系分解 — 概念之间的语法

一张好参考图的力量常常不在单个概念，而在概念之间的**关系**：

- **张力对**：粗粝材质 × 精致光线（对撞产生高级感）
- **配比**：90% 平静 + 10% 意外（意外太多变噪音）
- **依赖**：那种静谧只有在单一对角线构图下才成立

分解时额外输出 `relations: [{kind: tension|ratio|dependency, a, b, note}]`。融合不同 reference 的元素时，关系就是**兼容性语法**——不是把概念丢进袋子里摇，而是检查"这两个概念的组合破坏了谁的依赖、产生了什么新张力"。Critic 直接复用这些关系做审查。

## 三、品味画像 — soul 是"应然"，taste 是"实然"，差值最值钱

Soul 规定品牌应该是什么；你的每一次选择（校准游戏、采纳、差评、融合实验的去留）暴露你实际被什么吸引。积累后做聚类：

- **品味引力子（attractors）**："你被 'containment/收纳感' 吸引了 7 次，横跨 4 张不同来源的参考图"——你自己都未必意识到的偏好模式
- **Soul–Taste 差值报告**：品味画像和 soul 冲突的地方，要么 soul 该更新（你的品牌在进化），要么是漂移预警（你在偏离自己定的红线）。系统提出，你裁决。
- **目的条件化**：taste 不是全局的——hero 图的品味 ≠ 社媒图的品味。引力子按 purpose 分桶，概念 agent 按当前 purpose 加权选取。

## 四、Fusion Lab — 合成新 reference（自我繁殖回路）

**这是你点名要的核心**。新页面/新流程：

```
选 2-4 张概念卡（可跨 reference、可跨视角、可指定抽象层级）
  → 合成 agent 生成一张全新的纯美学 reference（无产品！只有美学）
  → 你评判：keep → 入库（source: 'synthesized'，初始低权重）/ discard
  → 入库的合成图同样被 decompose —— 新词汇进入词汇库
  → 下一代融合可以用上一代的产物
```

进化论结构完整：**交叉**（跨图重组）+ **变异**（Wildcard 对撞）+ **选择**（你的评判 + 采纳率）= 一个以你的品味为适应度函数的美学进化系统。原始 reference 是第 0 代；几轮之后，词汇库里会出现**任何单一来源都没有的视觉语言**——这就是"创造出新的 reference"。

防退化护栏：合成图必须标记代数（gen 1, gen 2…）；每一代必须通过 soul 红线检查才能入库；合成图权重只有被真实采纳（用它出的产品图被保存/导出）才会上升——防止美学近亲繁殖。

## 五、反品味 — 分解也要输出排斥

分解一张参考图时，同时提取"这张图做了什么是本品牌**绝不能做**的"（anti-concepts）。排斥和吸引一样是品味的一半。anti-concepts 直接喂给 Critic 的规则集，让评审会越来越懂你的底线。

## 实施顺序（建议）

| 期 | 内容 | 成本 | 为什么先/后 |
|---|---|---|---|
| R1 | **Fusion Lab**：选卡合成新 reference + 入库 + 再分解 + 代数标记 | 1 天 | 直击核心诉求，立刻可玩可验证 |
| R2 | **抽象阶梯**：decompose 升级四层 + 融合时选迁移层级 | 0.5 天 | 让 R1 的融合从模仿变创造 |
| R3 | **品味画像**：引力子聚类 + soul-taste 差值 + purpose 分桶 | 1 天 | 需要 R1/校准积累一点数据后更有料 |
| R4 | 关系语法 + 反品味 | 1 天 | 提升融合质量，等 R1 暴露问题后对症做 |
| R5 | 自动进化（定时自繁殖，预算受控） | 后置 | 等采纳率证明合成质量再自动化 |

## 边界提醒

1. 合成 reference 花的是真实生成费——Fusion Lab 走 flash 模型（$0.04/张），进化实验才不心疼。
2. 品味画像最少需要 ~20 个选择信号才有统计意义，前期显示"数据不足"比硬给结论诚实。
3. 不做的：跨品牌品味迁移、全自动无人评判的进化（你的裁决是适应度函数，去掉它系统就会漂）。
